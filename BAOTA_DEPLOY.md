# 宝塔面板部署指南

## 服务器准备

### 1. 登录阿里云服务器
- 使用SSH工具（如Xshell、Putty）登录您的阿里云服务器
- 确保服务器已安装宝塔面板
- 如果未安装宝塔面板，可执行以下命令安装：
  ```bash
  yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh
  ```

### 2. 登录宝塔面板
- 在浏览器中访问宝塔面板地址（通常为：http://服务器IP:8888）
- 使用初始账号密码登录（可在服务器终端查看）

## 项目部署

### 1. 创建网站
1. 登录宝塔面板后，点击左侧菜单「网站」→「添加站点」
2. 填写以下信息：
   - 域名：`tabify.top`
   - 备注：`Tabify静态网页部署服务`
   - 根目录：选择或创建一个目录（如 `/www/wwwroot/tabify`）
   - PHP版本：选择「纯静态」
   - 点击「提交」

### 2. 上传项目文件
1. 进入刚创建的网站根目录
2. 点击「上传」按钮，将本地项目文件（index.html、server.js、package.json）上传到服务器
3. 在服务器上创建必要的目录：
   ```bash
   mkdir -p uploads public
   ```

### 3. 安装Node.js
1. 点击左侧菜单「软件商店」→「运行环境」
2. 找到「Node.js版本管理」并安装
3. 安装完成后，选择一个Node.js版本（推荐16.x）并设置为默认

### 4. 安装项目依赖
1. 点击左侧菜单「终端」，进入网站根目录
2. 执行以下命令安装依赖：
   ```bash
   cd /www/wwwroot/tabify
   npm install
   ```

### 5. 配置PM2进程管理
1. 点击左侧菜单「软件商店」→「运行环境」
2. 找到「PM2管理器」并安装
3. 打开PM2管理器，点击「添加项目」
4. 填写以下信息：
   - 项目名称：`tabify-deploy`
   - 启动文件：`server.js`
   - 项目路径：`/www/wwwroot/tabify`
   - 点击「提交」

### 6. 配置Nginx反向代理
1. 回到网站管理页面，点击刚创建的网站「设置」
2. 点击「反向代理」→「添加反向代理」
3. 填写以下信息：
   - 代理名称：`tabify-proxy`
   - 目标URL：`http://127.0.0.1:3000`
   - 发送域名：`$host`
   - 点击「提交」

### 7. 配置SSL证书
1. 回到网站管理页面，点击「SSL」选项卡
2. 选择「Let's Encrypt」
3. 勾选需要申请证书的域名（包括 `tabify.top` 和 `*.tabify.top`）
4. 点击「申请」
5. 申请成功后，开启「强制HTTPS」

### 8. 配置DNS解析
1. 登录阿里云域名控制台
2. 找到 `tabify.top` 域名，点击「解析」
3. 添加以下记录：
   - 记录类型：A
   - 主机记录：@
   - 记录值：您的服务器IP
   - TTL：10分钟
   - 点击「确定」
4. 添加通配符记录：
   - 记录类型：A
   - 主机记录：*
   - 记录值：您的服务器IP
   - TTL：10分钟
   - 点击「确定」

### 9. 测试部署
1. 访问 `https://tabify.top` 查看部署界面
2. 上传HTML文件或文件夹测试部署功能
3. 访问生成的URL验证部署是否成功

## 常见问题排查

### 1. 服务无法启动
- 检查PM2状态，查看是否有错误信息
- 检查端口3000是否被占用
- 检查Node.js版本是否兼容

### 2. 上传失败
- 检查服务器磁盘空间是否充足
- 检查上传目录权限是否正确
- 检查Nginx配置是否正确

### 3. URL无法访问
- 检查DNS解析是否生效
- 检查SSL证书是否正确安装
- 检查Nginx反向代理配置是否正确

### 4. 安全配置
- 在宝塔面板中开启防火墙，只允许必要的端口（80、443、22）
- 定期更新服务器系统和软件
- 配置强密码和SSH密钥登录

## 维护命令

### 查看服务状态
```bash
pm run pm2 status
```

### 查看日志
```bash
pm run pm2 logs
```

### 重启服务
```bash
pm run pm2 restart all
```

### 停止服务
```bash
pm run pm2 stop all
```
